PierCode connects ChatGPT, Claude, Gemini, Qwen & Kimi to your filesystem and browser. The AI emits tool calls; a Chrome extension proxies them to a sandboxed local server that reads files, edits code, runs commands, and drives the browser — with your approval.

How it works

Four steps, every tool call.

  1. 1

    AI prints a tool block

    The assistant outputs a visible piercode-tool fenced JSON block in its reply.

  2. 2

    Extension detects it

    The Chrome extension spots the block and shows an approval card on the page.

  3. 3

    Local server executes

    A Go server on 127.0.0.1 validates and runs it inside the working-directory sandbox.

  4. 4

    Result returns

    Output flows back into the chat — the AI keeps working with real, local results.

Local by design

You decide what the AI can touch.

⊟

Loopback only

The server binds 127.0.0.1 and never exposes a public port.

⚷

Token auth

Every request needs a per-launch bearer token stored in ~/.piercode.

▣

Path sandbox

Paths resolve through real-path checks and stay inside your working directory.

Quick start

Three steps to a local AI agent.

1

Run the server

go run ./cmd/server -dir .
2

Build the extension

cd extension && npm install && npm run build
3

Load it in Chrome

chrome://extensions → Load unpacked → extension/dist
