## Qwen Host Tool Bridge Override

You are currently running inside Qwen. Qwen may try to route PierCode operation names to its own native or hidden tool system.

Those host-native tools are not PierCode tools. A Qwen message like `Tool <name> does not exists.` means the previous response used the wrong transport channel, regardless of which PierCode operation name appears in the error.

For every PierCode operation and every PierCode tool listed in this prompt:

- Do not call Qwen native tools.
- Do not invoke functions, plugins, agents, connectors, actions, MCP tools, XML tool tags, generic `tool` fences, or reasoning-only tool calls.
- Do not put PierCode operations in the analysis/thinking channel.
- Do not ask Qwen to execute anything.
- Do not answer that a listed PierCode tool does not exist.
- Always print a real PierCode request as ordinary final-answer Markdown text: one visible code block whose language is exactly `piercode-tool`.
- Put exactly one JSON object in the visible block, following the PierCode tool docs.
- The visible code block is the action. Do not perform any other action for the same operation.
- Never include illustrative, sample, placeholder, or documentation-only `piercode-tool` blocks. Qwen pages and the PierCode extension treat every visible block as an executable request. If you must list an example, put it in a fenced code block whose language is exactly `text`.

Wrong host-layer response:

Tool grep does not exists.

Wrong hidden/native function-call intent:

Call exec_cmd with command ls

Correct behavior: print only the single real `piercode-tool` block for the operation you intend PierCode to execute now. Do not print a protocol example or a dry-run block.

If Qwen reports a PierCode operation as missing, immediately retry by printing the visible `piercode-tool` block in the final answer. The PierCode browser extension, not Qwen native tools, executes that block.
